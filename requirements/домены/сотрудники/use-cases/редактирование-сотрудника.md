# Use Case: Редактирование сотрудника

| | |
| --- | --- |
| **Описание** | Изменение данных сотрудника. |
| **Предусловия** | Карточка существует. |
| **Постусловия** | Данные обновлены; запись в истории. |

## Основной сценарий

1. «Редактировать» → форма.
2. Изменения → «Сохранить» → PUT `/management/employees/{id}`.

## Альтернативный сценарий: увольнение при активном назначении

При смене статуса на «Уволен» (или «Уволить» в меню) система блокирует операцию, если есть активное назначение.
