# Use Case: Создание сотрудника

| | |
| --- | --- |
| **Описание** | Создание карточки сотрудника для последующих назначений. |
| **Предусловия** | Авторизация; список сотрудников. |
| **Постусловия** | Карточка создана (`status = ACTIVE`). |

## Основной сценарий

1. «Добавить сотрудника» → форма создания.
2. При необходимости тип «Внешний» → доп. поля.
3. Заполнение полей → «Создать» → POST `/management/employees`.

## Альтернативы

- Некорректные данные → сообщение об ошибке, исправление.
- «Отменить» → без сохранения.
