# Use Case: Создание проекта

| | |
| --- | --- |
| **Описание** | Менеджер создает проект для учета назначений и параметров проекта. |
| **Предусловия** | Авторизация; открыт раздел «Проекты». |
| **Постусловия** | Проект создан (`status = ACTIVE`). |

## Основной сценарий

1. «Создать проект» → форма создания.
2. Заполнить название, менеджера, тип, при необходимости код проекта и комментарий.
3. «Создать» → валидация → POST `/management/projects` → проект вверху списка.

## Альтернативы

- Не заполнены обязательные поля → подсветка, «Заполните все обязательные поля».
- Дубликат названия → «Проект с таким названием уже существует».
- «Отменить» / крестик → выход без сохранения.
