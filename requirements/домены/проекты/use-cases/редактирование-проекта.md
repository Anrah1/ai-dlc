# Use Case: Редактирование проекта

| | |
| --- | --- |
| **Описание** | Редактирование карточки проекта (из карточки или бокового меню списка). |
| **Предусловия** | Проект создан; статус не «Завершен» (для завершенных — предупреждение). |
| **Постусловия** | Изменения в БД и в истории изменений. |

## Основной сценарий

1. «Редактировать» → форма с текущими данными.
2. Изменения → «Сохранить».
3. Проверка: есть изменения, поля валидны, при смене статуса на «Завершен» — нет активных назначений.
4. PATCH `/management/project/{id}` → карточка проекта.

## Альтернативы

- Нет изменений → «Необходимо ввести хотя бы одно новое значение».
- Проект уже завершен → «Внимание! Проект уже завершен».
- «Отменить» → без сохранения.
