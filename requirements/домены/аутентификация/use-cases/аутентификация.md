# Use Case: Аутентификация

| | |
| --- | --- |
| **Функциональная область** | Аутентификация |
| **Описание** | Пользователь выполняет вход в систему, чтобы воспользоваться функционалом «Планер». |
| **Предусловия** | Выполнен переход по ссылке на систему; пароль получен у администратора. |
| **Постусловия** | Пользователь авторизован в системе. |
| **Роли** | Пользователь, система |

## Основной сценарий

1. Система отображает экран входа (Keycloak).
2. Пользователь вводит логин в поле «Username or email».
3. Пользователь вводит пароль в поле «Password».
4. Система выполняет аутентификацию:
   - при успехе — переход к п. 5;
   - при ошибке — **Альтернативный сценарий 1**.
5. Система выполняет авторизацию и открывает страницу, на которую пользователь изначально переходил.

## Альтернативный сценарий 1 — неверные учетные данные

1. Выполняются п. 1–4 основного сценария.
2. Система выводит сообщение: «Неверное имя пользователя или пароль».
3. Переход к п. 2 основного сценария.
