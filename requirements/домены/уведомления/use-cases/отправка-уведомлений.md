# Use Case: Отправка уведомлений

| | |
| --- | --- |
| **Описание** | Пользователь получает уведомления о событиях по назначениям в Telegram. |
| **Предусловия** | Пользователь в системе; аккаунт Telegram; бот подключен к профилю. |
| **Постусловия** | Сообщение доставлено в чат с ботом (или ошибка залогирована). |

## Основной сценарий

1. Сервис назначений по расписанию (3 шедулера) проверяет события.
2. При выполнении условия — сообщение в Kafka `notification-topic`.
3. Сервис нотификаций читает событие, запрашивает Chat ID в Keycloak.
4. Отправка сообщения в Telegram.

## События-триггеры

| Событие | Условие |
| --- | --- |
| Дата подключения | за 3 и 1 день до `start_date` |
| Дата отключения | за 30, 14, 7 и 1 день до `end_date` |
| Дата оплаты | в день `payment_date` |

## Матрица уведомляемых

| Событие | Лид | Менеджер | Руководители лида |
| --- | --- | --- | --- |
| Подключение | да | да | да |
| Отключение | да | да | да |
| Оплата | нет | да | нет |
