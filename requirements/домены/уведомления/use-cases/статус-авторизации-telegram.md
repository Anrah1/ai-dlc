# Use Case: Статус авторизации Telegram

##### Use case

| **Наименование** | Проверка статуса авторизации пользователя |
| --- | --- |
| **Функциональная область** | Сервис нотификаций |
| **Описание** | Я, как пользователь, хочу иметь возможность проверить свой статус авторизации в боте |
| **Предусловия** | - Пользователь зарегистрирован в системе. / - У пользователя есть активный аккаунт в Телеграм. |
| **Постусловия** | - Пользователь узнал статус входа в систему |
|  | Пользователь / Сервис нотификации = Телеграм бот / Сервис аутентификации  = Кейклок |

Основной сценарий 1. Проверка статуса авторизации пользователя
**Пользователь **нажимает кнопку "Меню".
**Сервис нотификации** бот возвращает список доступных команд.
**Пользователь** нажимает кнопку "/status".
**Сервис нотификации **отправляет Id пользователя в Сервис аутентификации
**Сервис аутентификации** ищет ChatId по данным пользователя.
Если ChatId найден переход к п.6 Основного сценария
Если ChatId не найден Переход к Альтернативному сценарию 1.
**Сервис аутентификации **возвращает статус "Авторизован**"**
**Сервис нотификации **отправляет сообщение "Вход выполнен. Твой username"
**Пользователь **получает сообщение со статусом Авторизован.
Альтернативный сценарий 1. ChatId не найден
**Сервис аутентификации **возвращает статус "Не авторизован**".**
**Сервис нотификации **отправляет сообщение "Вход не выполнен".
**Пользователь **получает сообщение со статусом Не авторизован.
