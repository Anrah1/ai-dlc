# Use Case: Создание назначения

| | |
| --- | --- |
| **Описание** | Пользователь назначает сотрудника на проект, чтобы отслеживать занятость и параметры подключения. |
| **Предусловия** | Пользователь авторизован; открыта вкладка «Назначения». |
| **Постусловия** | Создано назначение сотрудника на проект. |
| **Роли** | Пользователь, система |

## Основной сценарий

1. Пользователь нажимает «Создать назначение» / «Добавить».
2. Система открывает форму создания назначения.
3. Пользователь выбирает сотрудника (фамилия, имя).
4. Система определяет id и тип сотрудника:
   - если `type = "Внешний"` — отображает поля «Партнер» и «Ставка у партнера»;
   - иначе — переход к п. 5.
5. Система предзаполняет поля «Роль на проекте», «Подроль», «Грейд» из карточки сотрудника.
6. Пользователь при необходимости меняет роль, подроль, грейд, выбирает проект.
7. Пользователь заполняет заказчиков, ответственного за обратную связь, даты подключения и отключения, условие, даты оплаты.
8. При типе «Внешний» — заполняет партнера и ставку партнера.
9. Пользователь заполняет условия по ЧК (при необходимости несколько строк).
10. В зависимости от условия система показывает «Ставка без НДС» (Профитшеринг/Ставка) или «Проценты» (ЧК).
11. Пользователь заполняет ставку/проценты и при необходимости комментарий.
12. Пользователь нажимает «Создать».
13. Система проверяет поля:
    - пустые обязательные → «Заполните пустые поля»;
    - значения вне справочника → «Данные по \<поле\> не найдены»;
    - у сотрудника уже есть активное назначение → «Сотрудник уже назначен на проект. Вы уверены?» с кнопками «Посмотреть текущее назначение», «Да», «Изменить назначение»;
    - `start_date > end_date` → «Дата подключения не может быть позже даты отключения».
    - при ошибке пользователь исправляет данные (возврат к п. 12).
14. Система сохраняет запись (`status = ACTIVE`, POST `/management/appointments`), закрывает форму и показывает карточку назначения.

## Альтернативный сценарий — отмена

В любой момент пользователь нажимает «Отменить» — форма закрывается без сохранения, возврат на список назначений.
