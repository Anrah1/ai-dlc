# Use Case: Редактирование назначения

| | |
| --- | --- |
| **Описание** | Пользователь редактирует данные назначения для актуализации или исправления ошибок. |
| **Предусловия** | Пользователь авторизован; назначение создано. |
| **Постусловия** | Назначение обновлено; изменения зафиксированы в истории. |
| **Роли** | Пользователь, система |

## Основной сценарий

1. Пользователь выбирает назначение и нажимает «Редактировать».
2. Система открывает форму редактирования с текущими данными.
3. Пользователь изменяет поля.
4. Пользователь нажимает «Сохранить».
5. Система выполняет те же проверки, что при создании (обязательность, справочники, активное назначение, даты).
6. Система пишет аудит в историю изменений.
7. Система обновляет запись (PUT `/management/appointments/{id}`), закрывает форму и показывает карточку назначения.

## Альтернативный сценарий — отмена

«Отменить» — форма закрывается без сохранения.
