# Use Case: Завершение назначения

| | |
| --- | --- |
| **Описание** | Пользователь завершает назначение, чтобы в списке оставались только актуальные записи. |
| **Предусловия** | Пользователь авторизован; назначение создано. |
| **Постусловия** | Статус назначения «Завершено» или «Завершено, ожидает оплаты». |
| **Роли** | Пользователь, система |

## Основной сценарий

1. Пользователь выбирает назначение и нажимает «Завершить».
2. Система показывает диалог: «Вы уверены, что хотите завершить назначение?» — кнопки «Завершить», «Завершить, ожидает оплаты», «Отменить».
3. «Отменить» — диалог закрывается, карточка без изменений.
4. «Завершить» / «Завершить, ожидает оплаты»:
   - если статус уже завершенный — **альтернативный сценарий**;
   - иначе система меняет `status`, сохраняет в БД и обновляет карточку.

## Альтернативный сценарий — повторное завершение

Система показывает «Назначение уже завершено» / «Назначение уже завершено, ждет оплаты». Пользователь нажимает «А, точно» — уведомление закрывается.
