# Use Case: Отмена назначения

| | |
| --- | --- |
| **Описание** | Пользователь отменяет назначение, чтобы исключить его из актуальных. |
| **Предусловия** | Пользователь авторизован; назначение создано. |
| **Постусловия** | Статус назначения «Отменено». |
| **Роли** | Пользователь, система |

## Основной сценарий

1. Пользователь выбирает назначение и нажимает «Отменить».
2. Система показывает диалог: «Вы уверены, что хотите отменить назначение?» — «Да» / «Отменить».
3. «Отменить» в диалоге — закрытие без изменений.
4. «Да»:
   - если статус уже «Отменено» — **альтернативный сценарий**;
   - иначе система ставит `status = Отменено`, сохраняет и обновляет карточку.

## Альтернативный сценарий — повторная отмена

Уведомление «Назначение уже отменено» → «А, точно».
