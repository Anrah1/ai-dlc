# Импорт файла с вопросами-ответами кандидата на интервью

**PUT** `/management/candidates/{candidateId}/interview-results`

Описание
**Описание метода:**
Метод предназначен для загрузки файла Excel с вопросами и ответами кандидата на интервью и сохранением этих данных в БД.
Метод: PUT
Эндпоинт: https://management/candidates/{candidateId}/interview-results
**Строка** **запроса****:** PUT https://management/candidates/{candidateId}/interview-results
**Пример запроса:** PUT https://management/candidates/10/interview-results
**Параметры: **file (тип: file, обязательный параметр): Файл Excel, который пользователь отправляет через форму (multipart/form-data).
**Тело запроса: **Файл (бинарные данные).
**Лог****ика работы метода:**
Получение файла Excel через запрос.
Выполняется валидация файла и данных:
- проверить, что файл имеет допустимый формат (.xlsx, .xls);
- проверить, что файл не пустой;
- проверить файл на наличие двух столбцов.
Выполняется чтение файла.
Выполняется парсинг данных:
Считываются данные с каждой строки: questionText и answerText.
Выполняется валидация данных:
- проверить наличие пустых строк (пропускать пустые строки).
Данные сохраняются в БД:
Каждая пара questionText и answerText сохраняются в БД отдельной записью, где:

| **Название параметра** | **Тип данных** | **Обязательность** | **Описание** | **Мэппинг на БД** |
| --- | --- | --- | --- | --- |
| candidateId | number / (формат: integer ($int64)) | + | Идентификатор кандидата | planer.interview_results.candidate_id |
| questionText | string | + | Текст вопроса | planer.interview_results.question_text |
| answerText | string | - | Текст ответа | planer.interview_results.answer_text |

**Ответ:**
200 ОК. Выгрузка результатов прошла успешно.

| [ / { / "id": 1, / "candidateId": 10, / "questionText": "Что такое UC?", / "answerText": "Cценарное описание взаимодействия пользователя с программным продуктом для достижения конкретной цели." / }, / { / "id": 2, / "candidateId": 10, / "questionText": "Функции Use Case", / "answerText": "Документация требований, описание поведения системы, идентификация участников, планирование тестирования." / } / ] |
| --- |

**Ошибки:**
400 Bad Request: Некорректные данные запроса (отсутствует файл).

| { / "status": "error", / "message": "No file part" / } |
| --- |

или
400 Bad Request: Некорректные данные запроса (Файл не является Excel-файлом).

| { / "status": "error", / "message": "File type not allowed" / } |
| --- |

или
400 Bad Request: Некорректные данные запроса (Файл поврежден или пуст).

| { / "status": "error", / "message": "File is damaged or empty" / } |
| --- |

или
400 Bad Request: Некорректные данные запроса (Отсутствуют необходимые столбцы).

| { / "status": "error", / "message": "Missing required columns" / } |
| --- |

или
404 Not Found: Кандидат не найден.

| { / "message": "Кандидат с идентификатором 'id' не найден." / } |
| --- |
