# Редактирование карточки проекта

**PATCH** `/management/projects`

Редактирование карточки проекта
**Описание метода:**
Метод предназначен для редактирования карточки проекта.
**Логика выполнения метода:**
После получения данных выполняется валидация:
Проверка заполнения обязательных полей: id, name, manager_id, type, status. (не null)
Проверка уникальности названия проекта. иначе ошибка "409".
Проверка отсутствия активных назначений при  завершении проекта, иначе ошибка "422". Если у назначения статус canceled, то проект можно завершить.
Проверка наличия менеджера: если менеджер не найден ошибка "404".
Проверка при редактировании проекта по id, если проект не найден ошибка "404".
Проверка при создании аудита, если менеджер проекта не найден ошибка "404.
После успешной валидации выполняется сбор информации об изменении (аудит). Происходит сравнение старой записи с новыми полученными данными. В аудит входит:
Идентификатор изменяемой записи (project_id);
Название изменяемых полей (changed_field);
Тип изменяемых полей (field_data_type);
Старые и новые значения данных (previous_value и new_value);
Временная метка изменения (updated_at);
ФИ пользователя, внесшего изменения (updated_by).
Далее аудит отправляется в БД: схему history, таблицу Projects подробнее в описании .
3. Новые значения записи (полученные данные) обновляются в БД: схеме planer, таблице Projects.
Метод: PATCH
Эндпоинт: https://management/projects

**Валидации **
При получении запроса сервис проводит валидацию данных
Проверка заполнения обязательных полей: id, name, manager_id, type.
Проверка уникальности названия проекта. иначе ошибка "409".
Проверка отсутствия активных назначений при  завершении проекта, иначе ошибка "422". Если у назначения статус canceled, то проект можно завершить.
Проверка наличия менеджера: если менеджер не найден ошибка "404".
Проверка при редактировании проекта по id, если проект не найден ошибка "404".
Проверка при создании аудита, если менеджер проекта не найден ошибка "404.
Пример ошибки:

| { / "status": "422", / "message": "Нельзя завершить проект c активным назначением" / } |
| --- |
