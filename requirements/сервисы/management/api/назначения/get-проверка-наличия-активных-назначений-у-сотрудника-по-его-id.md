# Проверка наличия активных назначений у сотрудника по его ID

**GET** `/management/appointments/employees/{id}`

Получение информации о назначении по ID
**Описание метода:**
Метод проверяет наличие активных назначений у сотрудника.
Метод: GET
Эндпоинт: /management/appointments/employees/{id}
**Строка**** запроса****: **GET /management/appointments/employees/{id}
**Пример запроса:** GET /management/appointments/employees/1
**Лог****ика работы метода:**
Отправляется запрос в БД: поиск по сущности appointments, где employee_id = {id} (отправленный в запросе) и status = ACTIVE.
Если в ответе на запрос есть назначения, то в ответе метода отправляется isExists = true.
 Если в ответе нет значений, то в ответе метода отправляется isExists = false.
**Ответ:**

| **Параметр** | **Обязательность** | **Тип данных** | **Мэппинг на БД** | **Описание поля** |
| --- | --- | --- | --- | --- |
| isExists | + | boolean | - | Возвращает true, если у сотрудника есть активные назначения: возвращает false, если у сотрудника нет активных назначений |

**Пример успешного ответа: **
200 ОК:  Успешный ответ.

| { / "isExists": **true** / } |
| --- |
